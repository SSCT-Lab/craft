【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - PaddlePaddle 的 `paddle.bernoulli` 要求输入张量中的每个元素作为成功概率，必须满足 $ x_i \in [0, 1] $，而测试用例中传入了多个小于 0 的值（如 -0.926930），违反了该限制，因此抛出 `OutOfRange` 错误。
   - PyTorch 的 `torch.bernoulli` 允许输入任意实数，会先通过某种方式（如 clamp 到 [0,1]）处理非合法概率值，或直接容忍负值（实际行为为将负值视为 0），这与 PaddlePaddle 的严格校验行为存在本质差异。

3. 修复建议：
   - 该差异属于框架间对 Bernoulli 概率输入的合法性校验策略不同，属于可接受的行为差异；不建议提交 issue，但需在跨框架测试中注意对输入进行预处理以保证兼容性。
