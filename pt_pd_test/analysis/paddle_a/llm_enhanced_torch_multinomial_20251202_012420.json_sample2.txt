1. 结论标签：A 框架行为差异

2. 原因分析：  
   - PaddlePaddle 的 `paddle.multinomial` 要求输入张量中每个元素必须 ≥ 0，且不能全为 0（文档明确说明“Each element in x should be larger or equal to 0, but not all 0”），而输入中包含 -0.104454，违反了该约束。  
   - PyTorch 的 `torch.multinomial` 允许输入为负值，其行为是将输入视为未归一化的概率（即可解释为权重），并通过 softmax 处理后生成分布；而 PaddlePaddle 不支持此语义，要求输入必须为非负概率。

3. 修复建议：  
   - 此为框架间对输入语义的处理差异，属于合理的行为不一致，无需提交 issue；在迁移时需确保输入先经 softmax 归一化并保证非负性。
