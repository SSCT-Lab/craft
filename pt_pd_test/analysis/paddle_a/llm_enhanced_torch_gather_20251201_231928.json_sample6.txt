【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - PaddlePaddle 的 `paddle.gather` 要求 `index` 的秩（rank）必须为 0 或 1，且当 `index` 为二维时，其最后一个维度必须为 1（即形状应为 `[N, 1]`），但本例中 `index` 形状为 `[5, 10]`，最后一个维度为 10，违反了文档限制。
   - PyTorch 的 `torch.gather` 支持任意形状的 `index`（只要维度匹配），并允许在 `dim=0` 时使用二维索引（如 `[5,10]`），而 PaddlePaddle 在此场景下强制要求 `index` 最后一维为 1，导致报错。

3. 修复建议：
   - 该问题属于两框架在 `gather` 行为上的设计差异，非映射错误；由于 PaddlePaddle 当前实现对二维 `index` 有严格限制，而 PyTorch 允许更灵活的输入，建议提交 issue 至 PaddlePaddle 官方仓库，反映该接口与 PyTorch 不一致的行为。
