paddle_error 错误信息: (InvalidArgument) Cannot perform reduction along an axis (0) that has a size of 0.

【PyTorch 测试用例】
{
  "api": "torch.max",
  "input": {
    "shape": [
      0,
      2,
      3,
      4
    ],
    "dtype": "int64",
    "sample_values": []
  }
}

【PaddlePaddle 测试用例】
{
  "api": "paddle.max",
  "input": {
    "shape": [
      0,
      2,
      3,
      4
    ],
    "dtype": "int64",
    "sample_values": []
  }
}

【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - PaddlePaddle 的 `paddle.max` 在对空张量（shape 中包含 0）进行 reduction 操作时，会明确检查 reduce axis 所在维度的 size 必须大于 0，否则抛出 `Cannot perform reduction along an axis (0) that has a size of 0` 错误。
   - 而 PyTorch 的 `torch.max` 对于 shape 为 `[0, 2, 3, 4]` 的张量，在执行全局最大值（无 axis）或沿任意 axis 进行 reduction 时仍能正常处理，返回一个标量（如 `tensor(0)`），表现出对零维尺寸的容错性。

3. 修复建议：
   - 此为框架行为不一致问题，建议提交 issue 至 PaddlePaddle 官方仓库，指出其在处理零元素维度的 reduction 时过于严格，与 PyTorch 行为不一致，影响模型迁移兼容性。
