【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - PaddlePaddle 的 `paddle.bernoulli` 要求输入的每个元素作为概率值必须在 [0, 1] 区间内，而测试用例中传入了多个负值（如 -1.430141），违反了该限制，因此抛出 `OutOfRange: The probability should be >= 0` 错误。
   - PyTorch 的 `torch.bernoulli` 允许输入包含负值或大于 1 的值，会先对输入进行 clamp 到 [0, 1] 处理（即自动截断为合法概率），因此不会报错。

3. 修复建议：
   - 该差异属于框架语义不一致，建议提交 issue 至 PaddlePaddle 官方仓库，指出 `paddle.bernoulli` 对输入范围的严格校验与 PyTorch 不一致，且缺乏类似 `clamp` 的容错处理。
