【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - PaddlePaddle 的 `torch.masked_select` 要求 `mask` 必须是 `bool` 类型（即 `bool` tensor），但输入的 `mask` 为 `uint8`，尽管其值为 0 和 255（可视为布尔掩码），但类型不匹配。
   - 官方文档明确指出 `mask` 应为 `BoolTensor`，而当前传入的是 `uint8`，即使数值上可解释为真/假，类型不一致仍会触发类型校验失败。

3. 修复建议：
   - 该问题属于框架对 mask 类型严格校验的行为差异，建议提交 issue 给 PaddlePaddle 官方，反映其应支持非 bool 型掩码（如 uint8）的隐式转换以兼容 PyTorch 行为。
