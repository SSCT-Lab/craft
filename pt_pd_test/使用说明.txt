基于LLM的PyTorch与PaddlePaddle算子比较测试框架 - 使用说明
================================================================================

📁 文件说明
================================================================================

1. llm_enhanced_compare.py
   - 主程序文件
   - 包含完整的LLM增强测试框架
   - 约785行代码，包含详细中文注释

2. README_LLM_Enhanced.md
   - 详细的使用文档（英文标题，中文内容）
   - 包含功能介绍、使用方法、工作流程等

3. test_llm_enhanced.py
   - 测试脚本
   - 用于验证代码结构是否正确

4. 使用说明.txt
   - 本文件，快速上手指南

================================================================================
🚀 快速开始
================================================================================

步骤1：安装依赖
----------------
pip install pymongo torch paddle numpy pandas openai tqdm

步骤2：设置环境变量
------------------
Windows PowerShell:
    $env:DASHSCOPE_API_KEY="your_api_key_here"

Linux/Mac:
    export DASHSCOPE_API_KEY="your_api_key_here"

步骤3：确保MongoDB运行
---------------------
确保MongoDB服务正在运行，默认连接：mongodb://localhost:27017/
数据库名称：freefuzz-torch
集合名称：argVS

步骤4：运行测试
--------------
# 先运行基本测试，验证代码结构
python test_llm_enhanced.py

# 如果测试通过，运行主程序（默认测试torch.where的前2个用例）
python llm_enhanced_compare.py

================================================================================
⚙️ 自定义配置
================================================================================

修改测试算子和迭代次数：
------------------------
打开 llm_enhanced_compare.py，找到 main() 函数中的配置区域：

def main():
    # ==================== 测试参数配置 ====================
    operator_name = "torch.where"  # 修改这里：待测试的算子
    max_iterations = 3  # 修改这里：每个测试用例的最大迭代次数
    num_test_cases = 2  # 修改这里：要测试的用例数量，None表示测试所有用例
    # ====================================================
    
    # ... 其余代码保持不变

支持的算子示例：
- torch.where（默认，有4个测试用例）
- torch.nn.Dropout2d（有多个测试用例）
- torch.nn.Conv2d
- torch.nn.BatchNorm2d
- torch.nn.ReLU
- 等等（任何在MongoDB中有测试用例的算子）

参数说明：
- operator_name: 算子名称
- max_iterations: 每个测试用例的最大迭代次数（建议3-5次）
- num_test_cases: 要测试的用例数量
  * 设置为具体数字（如2）：只测试前2个用例
  * 设置为None：测试所有用例

================================================================================
📊 输出结果
================================================================================

控制台输出：
-----------
- 实时显示测试进度
- 每次迭代的执行结果
- LLM的分析和建议
- 最终测试摘要

JSON日志文件：
-------------
保存位置：pt_pd_log/ 目录
文件名格式：llm_enhanced_torch_nn_Dropout2d_YYYYMMDD_HHMMSS.json

日志内容包括：
- 算子名称
- 时间戳
- 总迭代次数
- 每次迭代的详细结果：
  * 测试用例信息
  * 执行结果（PyTorch和PaddlePaddle）
  * LLM操作建议

================================================================================
🔄 工作流程
================================================================================

1. 从MongoDB获取PyTorch算子测试用例
   ↓
2. 转换为PyTorch和PaddlePaddle可执行格式
   ↓
3. 执行测试用例，比较两个框架的结果
   ↓
4. 将执行结果提交给LLM（阿里千问大模型）分析
   ↓
5. LLM自主判断并返回操作建议：
   - 结果一致 → 进行变异（fuzzing）
   - 执行出错 → 进行修复
   - 结果不一致 → 判断是精度误差还是参数问题
   ↓
6. 使用LLM返回的新测试用例继续下一轮迭代
   ↓
7. 达到最大迭代次数或LLM建议跳过时结束
   ↓
8. 保存所有迭代结果到JSON文件

================================================================================
💡 核心特性
================================================================================

1. 自动测试用例迁移
   - 从PyTorch测试用例自动转换为PaddlePaddle版本
   - 处理参数名差异（如input→x）
   - 跳过不支持的参数（如inplace）

2. LLM智能分析
   - 使用阿里千问大模型（qwen-plus）
   - 自主判断测试结果
   - 生成修复或变异建议

3. 自动修复
   - 修正不合法的参数值
   - 调整输入张量的形状或类型
   - 移除不支持的参数

4. 智能变异（Fuzzing）
   - 修改输入张量的形状
   - 修改参数值（在合理范围内）
   - 添加或修改可选参数

5. 迭代优化
   - 支持多轮迭代
   - 每轮使用上一轮的结果
   - 持续改进测试用例

6. 详细日志
   - 完整的JSON格式日志
   - 包含所有迭代的详细信息
   - 便于后续分析和调试

================================================================================
⚠️ 注意事项
================================================================================

1. API密钥
   - 必须正确设置 DASHSCOPE_API_KEY 环境变量
   - 可以在阿里云DashScope控制台获取

2. MongoDB连接
   - 确保MongoDB服务正在运行
   - 确保数据库中有测试用例数据

3. 迭代次数
   - 建议设置为3-5次
   - 避免过多的LLM调用（会产生费用）

4. 网络连接
   - 需要稳定的网络连接以访问LLM API
   - 如果网络不稳定，可能导致LLM调用失败

5. 成本控制
   - LLM调用会产生费用
   - 注意控制测试规模
   - 可以先用少量迭代测试

================================================================================
🐛 故障排查
================================================================================

问题1：LLM调用失败
-----------------
解决方案：
- 检查API密钥是否正确
- 检查网络连接
- 查看控制台的详细错误信息

问题2：MongoDB连接失败
---------------------
解决方案：
- 确认MongoDB服务正在运行
- 检查连接URI是否正确（默认：mongodb://localhost:27017/）
- 检查数据库名称和集合名称

问题3：测试用例执行失败
----------------------
解决方案：
- 查看详细的错误信息
- 检查PyTorch和PaddlePaddle版本兼容性
- 确认算子名称是否正确

问题4：找不到算子
----------------
解决方案：
- 确认算子名称拼写正确
- 确认MongoDB中有该算子的测试用例
- 可以先查看数据库中有哪些算子
