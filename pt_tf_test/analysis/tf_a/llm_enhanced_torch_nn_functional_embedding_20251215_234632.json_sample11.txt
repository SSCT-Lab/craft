【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.embedding_lookup` 要求索引（indices）必须在 `[0, vocab_size)` 范围内，而测试用例中出现了 `-9` 和 `-3` 等负值索引，违反了该限制，导致运行时抛出 `indices[2,0] = -9 is not in [0, 1026)` 错误。
   - PyTorch 的 `torch.nn.functional.embedding` 允许负索引（如 -9），并将其视为无效索引处理（通常映射到 `padding_idx` 所指定的位置或默认为零向量），但 TensorFlow 无此容错机制，强制要求输入索引合法。

3. 修复建议：
   - 该行为差异属于框架设计层面的不一致，值得提交 issue 至 TensorFlow 官方仓库，建议增加对负索引的兼容性处理或明确文档说明。
