【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.embedding_lookup` 要求输入的 `ids` 必须为非负整数索引，且在 `[0, num_embeddings)` 范围内。错误信息中 `indices[0,1] = -6` 明确指出存在负值索引，而 TensorFlow 严格禁止此类输入，会直接抛出 `GatherV2` Op 错误。
   - 相比之下，PyTorch 的 `torch.nn.functional.embedding` 允许负索引（如 -6），并将其视为“未定义”或“填充”行为（若设置了 `padding_idx` 则可处理，否则仍会报错或产生无效输出），但其内部实现对负索引的处理方式与 TensorFlow 不同，导致行为不一致。

3. 修复建议：
   - 该问题属于框架间对负索引处理的语义差异，建议提交 issue 给 TensorFlow 官方，说明其 `embedding_lookup` 对负索引的强制限制与 PyTorch 不一致，可能影响模型迁移兼容性。
