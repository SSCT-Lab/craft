【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.sparse_softmax_cross_entropy_with_logits` 要求标签值（labels）必须在范围 [0, num_classes) 内，而错误信息显示存在多个非法标签值如 `-10`, `-9`, `-8`, `-7` 等，均小于 0，超出有效范围。
   - 尽管 PyTorch 的 `torch.nn.functional.cross_entropy` 支持通过 `ignore_index` 参数忽略特定值（如 -100），但 TensorFlow 该 API 并无等效的“忽略索引”机制；它直接要求标签为合法类别索引，任何越界值都会触发运行时错误。

3. 修复建议：
   - 此为框架设计层面的行为差异，非映射或参数问题，值得提交 issue 至 TensorFlow 官方以建议增强对无效标签的容错处理（如支持类似 ignore_index 的功能）。
