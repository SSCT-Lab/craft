   - TensorFlow 的 `tf.nn.sparse_softmax_cross_entropy_with_logits` 要求标签值（labels）必须在 `[0, num_classes)` 范围内，而错误日志中出现了大量超出范围的标签值（如 -10、-8、9、7 等），其中最大标签为 9，但类别数为 6（由 logits shape [1899, 6] 推断），因此标签值 6、7、8、9 及负值均无效。
   - 尽管 PyTorch 的 `nll_loss` 支持 `ignore_index`（如 -10）来忽略某些标签，但 TensorFlow 的该 API 无类似机制，且对标签范围有严格校验，不接受任何非 [0, 6) 的整数值。

3. 修复建议：
   - 此为框架行为差异，非映射或参数问题；由于 TensorFlow 未提供类似 `ignore_index` 的功能，应考虑在调用前对 labels 进行预处理过滤非法值，或改用组合方式实现等效逻辑。
