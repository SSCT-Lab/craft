【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.sparse_softmax_cross_entropy_with_logits` 要求标签值（labels）必须在范围 `[0, num_classes)` 内，且不允许出现负值或超出范围的索引。错误信息明确指出标签值 `-10` 无效，因为其不在 `[0, 10)` 范围内。
   - 尽管 PyTorch 的 `torch.nn.functional.cross_entropy` 支持通过 `ignore_index` 参数（如 `-10`）来忽略某些标签，但 TensorFlow 侧的该 API 并不支持“忽略索引”这一语义；它对所有标签值都要求为有效类别索引，否则直接抛错。

3. 修复建议：
   - 该问题属于框架间语义差异，应提交 issue 给 TensorFlow 社区以探讨是否可增加对 `ignore_index` 语义的支持，或明确文档说明此 API 不支持忽略非法标签。
