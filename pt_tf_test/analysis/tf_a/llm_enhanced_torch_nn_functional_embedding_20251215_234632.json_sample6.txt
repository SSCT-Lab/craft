【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：  
   - TensorFlow 的 `tf.nn.embedding_lookup` 要求输入索引（`ids`）中的每个值必须在 `[0, num_embeddings)` 范围内，否则会抛出 `indices[0,3] = -7 is not in [0, 1026)` 错误。而 PyTorch 的 `torch.nn.functional.embedding` 允许负索引（如 -7），并将其视为“未定义”或“忽略”，但不会直接报错（除非设置了 `padding_idx` 且该负值匹配）。  
   - 两者对负索引的处理策略不同：PyTorch 可容忍负索引（尤其当未显式设置 `padding_idx` 时），而 TensorFlow 严格要求索引非负且在范围内。

3. 修复建议：  
   - 由于该行为差异属于框架设计层面的语义不一致，且 TensorFlow 明确限制索引范围，建议在跨框架映射时增加预处理逻辑，将负索引转换为有效索引或显式屏蔽（如替换为 `padding_idx`），而非直接传递至 TensorFlow。
