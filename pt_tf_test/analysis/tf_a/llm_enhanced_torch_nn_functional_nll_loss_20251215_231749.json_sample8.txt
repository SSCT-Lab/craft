1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.sparse_softmax_cross_entropy_with_logits` 要求 `labels` 中的值必须在 `[0, num_classes)` 范围内，而错误信息显示存在大量超出范围的标签（如 -10、-9、8、9 等），其中最大值为 9，但 `num_classes=6`（由 logits shape [1899, 6] 推断），因此标签值 8、9 明显越界。
   - 尽管 PyTorch 的 `nll_loss` 允许通过 `ignore_index` 忽略某些无效标签（如 -1），但 TensorFlow 该 API 并无等效机制，且不支持忽略非法标签，一旦出现越界即抛出运行时错误。

3. 修复建议：
   - 此为框架间语义差异导致的行为不一致，非映射错误；建议提交 issue 给 TensorFlow 官方，指出其对 `sparse_softmax_cross_entropy_with_logits` 缺乏对非法标签的容忍机制，与 PyTorch 不一致。
