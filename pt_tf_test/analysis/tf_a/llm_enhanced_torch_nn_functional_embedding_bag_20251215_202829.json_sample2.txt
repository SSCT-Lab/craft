【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.embedding_lookup` 严格要求索引值（`ids`）必须在 `[0, num_embeddings)` 范围内，而测试用例中 `ids` 包含 `-3`、`-7`、`-9`、`-10` 等负值，且最大值为 `9`，超出权重矩阵维度 `8`，导致 `indices[1] = -3 is not in [0, 8)` 的错误。
   - PyTorch 的 `torch.nn.functional.embedding_bag` 支持负索引作为“未定义”或“忽略”处理（如通过 `padding_idx`），但其语义与 TensorFlow 严格边界检查不同，因此该行为差异属于框架对索引合法性的默认处理策略不一致。

3. 修复建议：
   - 该问题反映框架在索引有效性检查上的根本行为差异，值得提交 issue 给 TensorFlow 官方，说明其对负索引的严格限制与 PyTorch 的容错设计存在不一致。
