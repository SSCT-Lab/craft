【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.tensor_scatter_nd_update` 要求 `updates` 张量的形状必须与 `indices` 所指定的更新位置在“非索引维度”上完全匹配，即 `updates.shape` 必须等于 `indices.shape[:-1] + tensor.shape[indices.shape[-1]:]`。本例中 `indices.shape = [10, 2]`，`tensor.shape = [10, 5]`，因此 `updates` 应为 `[10, 3]`（因为第2维从索引位置取值后剩余维度是 `5-2=3`），但实际传入的是 `[10, 2]`，导致“Inner dimensions of output shape must match inner dimensions of updates shape”错误。
   - 与 PyTorch `torch.scatter` 行为不同：PyTorch 允许 `src` 与 `index` 在非 `dim` 维度上不一致，只要 `index` 和 `src` 的前缀维度一致即可（如 `[10,2]` 可用于更新 `[10,5]` 中第1维的两个位置），而 TensorFlow 的 `tensor_scatter_nd_update` 对更新数据的“内层维度”有严格对齐要求。

3. 修复建议：  
   该问题源于框架间语义差异，且 TensorFlow 的设计限制明确，不属于映射或参数错误，因此无需修改测试用例或映射逻辑；建议提交 issue 至 TensorFlow 官方仓库，说明其 `tensor_scatter_nd_update` 与 PyTorch `scatter` 在多维更新时的行为不一致，可能影响模型迁移。
