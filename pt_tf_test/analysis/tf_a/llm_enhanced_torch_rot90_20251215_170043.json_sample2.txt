【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.image.rot90` 要求输入张量的前三个维度（即空间维度）必须大于 0，而本例中输入 shape 为 `(1, 0, 4, 4)`，第二维为 0，违反了该 API 的输入约束。
   - 根据 TensorFlow 官方文档，`tf.image.rot90` 用于图像旋转，其输入被设计为具有有效高度和宽度的图像张量，因此对非空形状有明确要求；而 PyTorch 的 `torch.rot90` 允许任意维度中包含 0（如空通道），并能正常处理此类张量。

3. 修复建议：
   - 该问题反映了框架间在空维度处理上的行为差异，值得提交 issue 至 TensorFlow 官方，以明确 `tf.image.rot90` 是否应放宽对中间维度为 0 的限制，或增加更清晰的错误提示。
