【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.searchsorted` 要求 `sorted_sequence` 必须是严格递增的序列，但测试用例中传入的 `sorted_sequence`（shape=[100]）包含非单调值（如 `-1.794...`, `-0.592...`, `1.456...` 等），违反了该 API 的前置条件，导致执行时触发 `Leading dim_size of both tensors must match` 错误（实际为内部实现中对边界排序的校验失败，错误信息虽不精确但指向维度匹配问题）。
   - 与之对比，PyTorch `torch.bucketize` 并不要求 `boundaries` 严格递增，仅在文档中说明“若未严格递增，返回值未定义”，因此在语义上容忍度更高，不会直接报错。

3. 修复建议：
   - 该问题属于框架行为差异，可提交 issue 至 TensorFlow 官方仓库，指出 `tf.searchsorted` 在输入非严格递增时应显式抛出更准确的错误（如“sorted_sequence must be strictly increasing”），而非误导性地提示维度不匹配。
