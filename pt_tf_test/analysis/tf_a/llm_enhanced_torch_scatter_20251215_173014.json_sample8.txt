【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.tensor_scatter_nd_update` 要求 `updates` 的形状必须与 `indices` 索引出的子张量形状一致，即 `updates.shape` 应为 `[2, 5]`（因为 `indices` 是 `[2, 1]`，对应第1维索引，输出是 `[10, 5]`，所以每个更新应保持5个元素），但实际传入的 `updates` 形状为 `[2]`，导致内层维度不匹配。
   - 而 PyTorch `torch.scatter` 在 `dim=1` 时允许 `src` 形状为 `[2, 1]`，通过广播扩展到 `[2, 5]`，这与 TensorFlow 的严格形状对齐要求不同，属于框架间行为差异。

3. 修复建议：
   - 此为框架实现差异，非映射错误，建议提交 issue 至 TensorFlow 官方，指出 `tensor_scatter_nd_update` 在处理非完整维度更新时缺乏广播支持，与 PyTorch 行为不一致。
