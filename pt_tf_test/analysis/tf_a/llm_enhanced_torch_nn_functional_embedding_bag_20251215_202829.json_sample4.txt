【LLM 分析结果】
1. 结论标签：A 框架行为差异

2. 原因分析：
   - TensorFlow 的 `tf.nn.embedding_lookup` 要求 `ids` 中的索引值必须在 `[0, num_embeddings)` 范围内，而测试用例中 `ids` 包含 `-1` 和 `-7`，违反了该限制，导致运行时抛出 `indices[6] = -1 is not in [0, 8)` 错误。
   - 尽管 PyTorch 的 `torch.nn.functional.embedding_bag` 支持负索引（如 `-1`）并将其视为 `padding_idx`（若显式指定），但 TensorFlow 的 `embedding_lookup` 无此语义支持，且不接受负索引。

3. 修复建议：
   - 此为框架间行为差异，应提交 issue 至 TensorFlow 官方仓库，说明其 `embedding_lookup` 不支持负索引，而 PyTorch 对应 API 可处理此类情况，建议增加对 `padding_idx` 类似语义的支持或明确文档警告。
