# Fuzzing 差分测试修复记录 (2026-01-31)

## 问题描述

在运行 `llm_fuzzing_diff_test_concurrent.py` 时发现两个主要问题：

### 问题 1: LLM 生成的测试用例缺少必需参数

**现象**：
```json
"torch_error": "TypeError: vdot() missing 1 required positional arguments: \"other\""
```

**原因**：
- LLM 在生成变异测试用例时，没有保留原始用例中的必需参数（如 `other`, `dim`, `axis` 等）
- 导致 PyTorch 或 TensorFlow 执行时因缺少参数而失败

### 问题 2: 比较函数产生假阳性（False Positive）

**现象 1 - 空容器误判**：
```json
"comparison_error": "结果不一致: torch=(), tf=[]"
```
- PyTorch 返回空 tuple `()`
- TensorFlow 返回空 list `[]`
- 实际上两者都是空的，应该判定为一致

**现象 2 - NaN 误判**：
```json
"comparison_error": "结果不一致: torch=nan, tf=nan"
```
- 两个框架都返回 NaN
- 由于 `nan != nan` 的特性，被误判为不一致

## 修复方案

### 修复 1: 增强 LLM 提示词

在 `build_fuzzing_prompt()` 函数中添加了明确的参数完整性要求：

```python
【关键要求：参数完整性】
**必须包含原始测试用例中的所有必需参数！**
- 如果原始用例有 `other` 参数，变异后的用例也必须有 `other` 参数
- 如果原始用例有 `dim`/`axis` 参数，变异后的用例也必须有对应参数
- 如果原始用例有其他配置参数（如 `kernel_size`, `stride` 等），变异后也必须保留
- **不要遗漏任何必需参数，否则会导致执行失败！**
```

同时在 JSON 示例中明确展示了如何保留参数：

```json
{
  "torch_test_case": {
    "api": "torch.vdot",
    "input": {...},
    "other": {...},      // 保留 other 参数
    "dim": ...,          // 保留 dim 参数
    "其他必需参数": ...
  },
  "tensorflow_test_case": {
    "api": "tf.tensordot",
    "input": {...},
    "other": {...},      // 保留 other 参数
    "axis": ...,         // 保留 axis 参数
    "其他必需参数": ...
  }
}
```

### 修复 2: 完善比较函数

在 `compare_results()` 函数中添加了多种特殊情况处理：

#### 2.1 标量 NaN 处理
```python
# 处理标量 NaN 的情况
if isinstance(torch_res, (float, np.floating)) and isinstance(tf_res, (float, np.floating)):
    if np.isnan(torch_res) and np.isnan(tf_res):
        comparison["results_match"] = True
        return comparison
```

#### 2.2 标量 Inf 处理
```python
elif np.isinf(torch_res) and np.isinf(tf_res):
    if np.sign(torch_res) == np.sign(tf_res):
        comparison["results_match"] = True
    else:
        comparison["comparison_error"] = f"Inf 符号不一致: torch={torch_res}, tf={tf_res}"
    return comparison
```

#### 2.3 空容器处理（tuple vs list）
```python
# 处理空容器的情况（tuple vs list）
if isinstance(torch_res, (tuple, list)) and isinstance(tf_res, (tuple, list)):
    if len(torch_res) == 0 and len(tf_res) == 0:
        comparison["results_match"] = True
        return comparison
```

#### 2.4 容器递归比较
```python
# 递归比较容器中的每个元素
if len(torch_res) != len(tf_res):
    comparison["comparison_error"] = f"容器长度不一致: torch={len(torch_res)}, tf={len(tf_res)}"
    return comparison

all_match = True
for i, (t_item, tf_item) in enumerate(zip(torch_res, tf_res)):
    # 递归调用比较函数
    item_comparison = compare_results(
        {"success": True, "result": t_item, "shape": None, "dtype": None, "error": None},
        {"success": True, "result": tf_item, "shape": None, "dtype": None, "error": None},
        rtol, atol
    )
    if not item_comparison["results_match"]:
        all_match = False
        comparison["comparison_error"] = f"容器第 {i} 个元素不一致: {item_comparison.get('comparison_error', '未知原因')}"
        break

if all_match:
    comparison["results_match"] = True
```
